
## Label pipeline for PTPublish 

The complete picture of how a label is created and consumed by a PTPublish job:


### 1. Where the label is created — `MediaComposerService`

The label is generated automatically as part of the normal media composition pipeline, **before** any writer service is invoked. No extra steps are needed in `PTPublishService`.

| Service | What it does |
|---|---|
| `MediaComposerService` | Builds the fileset, then calls `LabelCreator.createLabel()` using Apache FOP + the `label.xsl` stylesheet. The resulting file is stored in the SpoolDir `label/` subdirectory and attached to the `MediaCreationRequest` via `rq.setLabelFile(f)`. |
| `PTPublishService.writeJrqFile()` | Already reads `rq.getLabelFile()` and, if it exists, writes `PrintLabel = <path>` into the `.jrq` file sent to PTBurn. |

### 2. Enabling label creation (one-time JMX setup)

In the JMX console at `http://localhost:8080/jmx-console`, open **`dcm4chee.cdw:service=MediaComposer`** and set:

| Attribute | Value |
|---|---|
| `LabelFileFormat` | `PNG`, `TIFF`, `PDF`, or `PS` (currently `NO` = disabled) |
| `LabelResolution` | e.g. `300` (dpi, only relevant for PNG/TIFF) |
| `ConfigDir` | Must point to a directory containing `label.xsl` — the config dir ships at `server/default/conf/dcm4chee-cdw/` |

### 3. What the XSL stylesheet controls

`label.xsl` (at label.xsl) is the FOP template that determines the visual layout. Edit it to customise what appears on the label (patient name, study date, volume name, etc.).

### 4. What PTBurn does with it

When `PrintLabel = <path>` appears in the `.jrq` file, PTBurn reads that image/PDF and prints it to the robot's disc printer inline, without the `LabelPrintService` being involved at all. The `LabelPrintService` (Java print subsystem) is an alternative path used by CDRecord/NeroCmd — for PTPublish, PTBurn handles printing natively.

### 5. Keeping the label file

The label file is deleted after the job by default. To keep it on the host, set **`KeepLabelFiles`** to `YES` on the PTPublish writer MBean (`dcm4chee.cdw:service=MediaWriter,name=PTPublish`) in the JMX console.

Where ISO/label files originate

ISO files are produced via spool/fileset workflow under JBoss data paths from dcm4chee-cdw-spooldir-xmbean.xml (default fileset directory).
Label files are generated by MediaComposer/LabelCreator and stored under SpoolDir label path from dcm4chee-cdw-spooldir-xmbean.xml (LabelDirPath, default label), then attached to request and consumed by PTPublish in PTPublishService.java.
Shared folders PTBurn needs (conceptually)

Job folder share: PTBurn watches this for .jrq, writes .don/.err.
Status share: PTBurn writes/updates SystemStatus.txt and <RobotName>.txt.
ISO share: location containing the ISO file referenced by ImageFile=....
Label share: location containing label file referenced by PrintLabel=....

DICOM CDW Server

Product Brochure — Hybrid (Business + Technical)

dcm4chee-cdw is a Java-based implementation of a DICOM Media Creation Management Server. It receives imaging content, composes standards-oriented media outputs, generates optional web and label artifacts, and supports practical operational handoff to writer/publish workflows.

Executive Value

  • Automates media creation tasks that are often manual and error-prone.
  • Provides repeatable, inspectable workflow stages for imaging operations teams.
  • Supports interoperability requirements where physical or packaged media remains necessary.
  • Allows modernization (containerization) without discarding proven operational patterns.

What It Does

  1. Accepts DICOM objects and media creation requests.
  2. Composes filesets and optional web content.
  3. Generates ISO images and optional labels.
  4. Passes jobs to downstream writer/publish services.
  5. Maintains queue-based status visibility and operational control.

Key Capabilities

DICOM Media Creation Management

  • Implements core behavior of a DICOM media creation server.
  • Supports patient/study-driven packaging patterns.
  • Designed for asynchronous, service-based execution.

Label and Web Output Customization

  • label.xsl drives output layout for media labels.
  • common.xsl centralizes reusable formatting templates.
  • web.xsl + related XSL files generate browsable media content.

PT Publish Workflow Alignment

  • Supports share-folder integration for job/status exchanges.
  • Fits operational pipelines where robotic burn/print systems are used.
  • Encourages deterministic handoff via generated request artifacts.

Reference Deployment

  • DICOM CDW service exposed on DICOM and HTTP/JMX ports.
  • Mounted volumes for job, status, ISO, and label data exchange.
  • Optional integration with external writer/robot systems.

Operational Controls

  • JMX-exposed services and attributes for runtime tuning.
  • Configurable JVM settings via JAVA_OPTS.
  • Health-check friendly container lifecycle support.
  • Template-level control over generated user-facing artifacts.

Security & Reliability Recommendations

  • Restrict administrative interfaces and segment DICOM traffic.
  • Use controlled access for shared folders and generated artifacts.
  • Harden host/container baseline and capture queue/write failures in monitoring.
  • Define backup/retention policies for ISO, label, and job audit artifacts.

Ideal Use Cases

  • Radiology departments needing dependable CD/DVD packaging workflows.
  • Interop teams requiring fallback distribution beyond network exchange.
  • Organizations maintaining mixed legacy systems while adopting container operations.
  • Integrators that need transparent, configurable media pipelines.
